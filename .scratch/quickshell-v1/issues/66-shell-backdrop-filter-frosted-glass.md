## What to build

The **shell** rim is drawn on a `<canvas>` with semi-transparent `theme.palette.surface` fills (`shell-animation.ts` → `themeToShellColors`). It tints over the background but does **not** blur it — unlike launcher/settings overlays which use `backdrop-filter`. The shell should use real **frosted glass**: high-radius `backdrop-filter: blur()` with tint derived from the workspace image/palette.

Reference technique: [Josh W. Comeau — backdrop-filter frosted glass](https://www.joshwcomeau.com/css/backdrop-filter/) — extended backdrop child (200% height), `mask-image` trim, `pointer-events: none`, optional edge layer, semi-opaque tint from palette.

Implementation may require hybrid DOM backdrop layer + canvas notch path, or replacing canvas fill with masked backdrop divs aligned to the shell geometry. ADR if paint model changes.

## Acceptance criteria

- [ ] Shell rim blurs the canvas/background image behind it at rest (not just surface color alpha)
- [ ] Blur radius and tint feel premium on starter themes (human sign-off)
- [ ] Notch expansion still deforms correctly; no pointer-event regressions on handles/menus
- [ ] Graceful fallback when `backdrop-filter` unsupported (opaque surface acceptable)
- [ ] Performance acceptable on typical laptop display (no jank on workspace switch)
